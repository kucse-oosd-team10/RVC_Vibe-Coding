# RVC Software Requirements Specification

## 1. 현재 단계

현재 단계는 Requirements Analysis이다. 이 문서는 RVC(Robot Vacuum Cleaner)의 `rvc-controller`에 대한 자동 진공 청소 제어 로직 요구사항을 정리한다.

## 2. 목적

본 프로젝트의 목적은 하드웨어 세부 구현에 의존하지 않는 `rvc-controller`의 자동 진공 청소 제어 로직 요구사항을 정의하는 것이다.

## 3. 범위

### 3.1 시스템 바운더리

본 프로젝트에서 개발할 시스템의 바운더리는 `rvc-controller`이다. 실제 센서 하드웨어와 실제 actuator는 `rvc-controller` 외부 actor로 정의한다.

`rvc-controller`는 다음 외부 actor와 인터페이스를 가진다.

| 외부 actor | `rvc-controller` 관점의 역할 |
|---|---|
| 사용자 | 청소 시작 및 종료 버튼 입력을 제공한다. |
| 전방 센서 하드웨어 | 전방 장애물 감지 interrupt 입력을 제공한다. |
| 측면 센서 하드웨어 | 좌측 및 우측 장애물 감지 polling 입력을 제공한다. |
| 먼지 센서 하드웨어 | 먼지 감지 입력을 제공한다. |
| Movement Motor | 전진, 정지, 후진, 좌회전, 우회전 명령을 수신하는 외부 actuator이다. |
| Cleaning Motor | OFF, NORMAL, POWER_UP 흡입 명령을 수신하는 외부 actuator이다. |

### 3.2 포함 범위

- 사용자 버튼 입력에 의한 청소 시작 및 종료
- 자동 진공 청소 중 기본 전진 동작
- 전방, 좌측, 우측 장애물 감지에 따른 회피 동작
- 먼지 감지에 따른 진공 흡입 상태 제어
- 전방/측면 센서, movement motor, cleaning motor와의 추상 인터페이스 요구사항
- 하드웨어 독립적인 제어 로직 요구사항

### 3.3 제외 범위

- 실제 센서 하드웨어 내부 동작
- 실제 actuator 내부 동작
- 실제 모터 구동 회로 및 물리 제어 세부사항
- 지도 작성, 경로 최적화, 공간 커버리지 알고리즘
- 배터리 관리, 충전, 도킹
- 앱 연동 및 사용자 인터페이스
- 후방 장애물 감지
- 후진 최대 허용 시간 처리
- 360도 회전 탈출 기능

## 4. 정의

| 용어 | 의미 |
|---|---|
| RVC | Robot Vacuum Cleaner |
| rvc-controller | 본 프로젝트에서 개발할 제어 시스템 |
| 외부 actor | `rvc-controller` 바운더리 밖에 존재하며 입력을 제공하거나 명령을 수신하는 외부 요소 |
| actuator | `rvc-controller`의 명령을 받아 물리 동작을 수행하는 외부 장치 |
| OFF | 전체 시스템 전원이 꺼진 상태 |
| NORMAL | 전원 ON 상태에서의 기본 진공 흡입 상태 |
| POWER_UP | 먼지 감지 시 3초 동안 유지되는 강화 진공 흡입 상태 |
| 전방 장애물 | 전방 센서로 감지되는 장애물 |
| 좌측 장애물 | 좌측 센서로 감지되는 장애물 |
| 우측 장애물 | 우측 센서로 감지되는 장애물 |

## 5. 가정

- 청소 시작 및 종료는 사용자 버튼 입력으로 발생한다.
- `rvc-controller`는 실제 센서 하드웨어와 actuator 자체를 구현하지 않는다.
- 전방 센서 하드웨어는 interrupt 방식으로 입력을 제공한다.
- 측면 센서 하드웨어는 좌측 및 우측 장애물 값을 polling 방식으로 제공한다.
- 후진 중 후방 장애물은 발생하지 않는다고 가정한다.
- 회전 모드 진입 후 10초가 지나면 회전이 성공적으로 완료된 것으로 간주한다.
- 후진 중 좌우 센서 polling 주기는 추후 정의한다.
- 좌회전은 기본 우선 회전 방향이다.

## 6. 기능 요구사항

| ID | 요구사항 |
|---|---|
| RVC-FR-001 | RVC는 사용자 청소 시작 버튼 입력을 받으면 자동 진공 청소를 시작해야 한다. |
| RVC-FR-002 | RVC는 사용자 청소 종료 버튼 입력을 받으면 청소 동작을 종료하고 OFF 상태로 전환해야 한다. |
| RVC-FR-003 | RVC는 청소 중 기본적으로 전진해야 한다. |
| RVC-FR-004 | RVC는 전방 센서 하드웨어의 장애물 입력을 회피 판단의 우선 입력으로 사용해야 한다. |
| RVC-FR-005 | 전방 장애물이 감지되지 않으면 RVC는 좌측 및 우측 장애물 센서 값을 회피 판단에 사용하지 않아야 한다. |
| RVC-FR-006 | RVC는 전방 장애물을 interrupt 방식 입력으로 받아야 한다. |
| RVC-FR-007 | RVC는 좌측 및 우측 장애물을 polling 방식 입력으로 받아야 한다. |
| RVC-FR-008 | 전진 중 전방 장애물만 감지되면 RVC는 정지한 후 좌회전해야 한다. |
| RVC-FR-009 | 전방 장애물과 좌측 장애물이 감지되고 우측 장애물이 감지되지 않으면 RVC는 정지한 후 우회전해야 한다. |
| RVC-FR-010 | 전방 장애물과 우측 장애물이 감지되고 좌측 장애물이 감지되지 않으면 RVC는 정지한 후 좌회전해야 한다. |
| RVC-FR-011 | 전방, 좌측, 우측 장애물이 모두 감지되면 RVC는 정지한 후 후진해야 한다. |
| RVC-FR-012 | 후진 중 좌측 장애물이 감지되지 않게 되면 RVC는 좌회전해야 한다. |
| RVC-FR-013 | 후진 중 우측 장애물이 감지되지 않게 되면 RVC는 우회전해야 한다. |
| RVC-FR-014 | 후진 중 좌측 및 우측 장애물이 동시에 감지되지 않게 되면 RVC는 좌회전해야 한다. |
| RVC-FR-015 | RVC는 회전 모드 진입 후 10초가 지나면 회전이 완료된 것으로 간주해야 한다. |
| RVC-FR-016 | 회전 완료 후 RVC는 다시 전진해야 한다. |
| RVC-FR-017 | RVC가 정지, 회전, 후진 중일 때 진공 흡입 상태는 NORMAL이어야 한다. |
| RVC-FR-018 | 정지, 회전, 후진 중 먼지가 감지되면 RVC는 즉시 POWER_UP으로 전환하지 않아야 한다. |
| RVC-FR-019 | 정지, 회전, 후진 중 감지된 먼지는 이후 전진 시작 시 POWER_UP 진입 조건으로 사용되어야 한다. |
| RVC-FR-020 | OFF 상태에서는 자동 청소 제어 로직과 진공 흡입 동작이 수행되지 않아야 한다. |
| RVC-FR-021 | 사용자 청소 시작 입력이 발생하면 RVC는 OFF에서 전원 ON 상태로 전환되어야 한다. |
| RVC-FR-022 | 전원 ON 상태 진입 시 진공 흡입 상태는 NORMAL이어야 한다. |
| RVC-FR-023 | 전원 ON 상태에서 RVC가 전진 중일 때 진공 흡입 상태는 NORMAL 또는 POWER_UP 중 하나여야 한다. |
| RVC-FR-024 | 전진 중 먼지가 감지되지 않으면 진공 흡입 상태는 NORMAL이어야 한다. |
| RVC-FR-025 | 전진 중 먼지가 감지되면 진공 흡입 상태는 POWER_UP으로 전환되어야 한다. |
| RVC-FR-026 | POWER_UP 상태는 먼지 감지 후 3초 동안 유지되어야 한다. |
| RVC-FR-027 | POWER_UP 상태에서 3초가 경과하면 RVC는 먼지 감지 값을 다시 확인해야 한다. |
| RVC-FR-028 | 3초 경과 후 먼지가 다시 감지되면 RVC는 POWER_UP 상태를 유지하고 3초 타이머를 다시 시작해야 한다. |
| RVC-FR-029 | 3초 경과 후 먼지가 감지되지 않으면 RVC는 NORMAL로 복귀해야 한다. |
| RVC-FR-030 | `rvc-controller`는 movement motor에 전진, 정지, 후진, 좌회전, 우회전에 해당하는 추상 이동 명령을 제공해야 한다. |
| RVC-FR-031 | `rvc-controller`는 cleaning motor에 OFF, NORMAL, POWER_UP에 해당하는 추상 흡입 명령을 제공해야 한다. |
| RVC-FR-032 | 청소 시작 시 `rvc-controller`는 전방 센서 하드웨어, 측면 센서 하드웨어, 먼지 센서 하드웨어와의 센서 인터페이스를 초기화해야 한다. |
| RVC-FR-033 | 청소 종료 시 `rvc-controller`는 전방 센서 하드웨어, 측면 센서 하드웨어, 먼지 센서 하드웨어와의 센서 인터페이스를 종료해야 한다. |

## 7. 비기능 요구사항

| ID | 요구사항 |
|---|---|
| RVC-NFR-001 | `rvc-controller`는 특정 하드웨어 장치나 제조사 API에 의존하지 않아야 한다. |
| RVC-NFR-002 | 요구사항은 향후 설계, 구현, 테스트와 추적 가능해야 한다. |
| RVC-NFR-003 | 자동 청소 제어 로직은 센서 입력, 판단 로직, actuator 명령의 책임을 구분할 수 있도록 정의되어야 한다. |

## 8. 제약사항

| ID | 제약사항 |
|---|---|
| RVC-CON-001 | 전방 장애물 입력은 interrupt 방식이라는 전제를 따른다. |
| RVC-CON-002 | 좌측 및 우측 장애물 입력은 polling 방식이라는 전제를 따른다. |
| RVC-CON-003 | 회전 완료 시간은 현재 요구사항 기준 10초로 정의한다. |
| RVC-CON-004 | POWER_UP 유지 시간은 3초로 정의한다. |
| RVC-CON-005 | 후방 장애물 감지는 현재 범위에서 제외한다. |
